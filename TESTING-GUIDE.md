# Sample Testing Examples for Spring Boot Scheduling Service\n\nThis directory contains comprehensive examples and test cases for the Spring Boot Scheduling Application.\n\n## 📋 Contents\n\n### 1. **SampleDataGenerator.java**\nAutomatically generates sample data on application startup.\n\n**Features:**\n- Creates 5 sample reports with different statuses (PENDING, IN_PROGRESS, COMPLETED, FAILED)\n- Creates 6 sample tasks with different execution frequencies (DAILY, WEEKLY, MONTHLY, HOURLY)\n- Displays comprehensive statistics on startup\n- Logs all operations for debugging\n\n**How to Use:**\n1. Run the application: `mvn spring-boot:run`\n2. Sample data is automatically generated on startup\n3. Check logs for confirmation\n\n### 2. **ApiIntegrationTests.java**\nComprehensive JUnit 5 integration tests using RestAssured.\n\n**Test Coverage:**\n- ✅ 15+ Report endpoint tests\n- ✅ 10+ Task endpoint tests\n- ✅ Status filtering tests\n- ✅ CRUD operation tests\n- ✅ Statistics endpoint tests\n\n**How to Run Tests:**\n```bash\n# Run all tests\nmvn test\n\n# Run specific test class\nmvn test -Dtest=ApiIntegrationTests\n\n# Run specific test method\nmvn test -Dtest=ApiIntegrationTests#testGetAllReports\n```\n\n### 3. **test-api.sh**\nBash script with 25+ cURL command examples for manual API testing.\n\n**Features:**\n- Color-coded output for easy reading\n- Covers all endpoints (Reports and Tasks)\n- Includes various HTTP methods (GET, POST, PUT)\n- Shows HTTP status codes\n\n**How to Use:**\n```bash\n# Make script executable\nchmod +x scripts/test-api.sh\n\n# Run all tests\n./scripts/test-api.sh\n\n# Or run individual commands\ncurl -X GET http://localhost:8080/api/reports\n```\n\n### 4. **Postman Collection**\nReadily importable Postman collection with all API endpoints.\n\n**Features:**\n- Pre-configured request templates\n- Sample JSON payloads\n- Environment variables ready\n- Organized into folders (Reports, Tasks)\n\n**How to Use:**\n1. Open Postman\n2. Click \"Import\"\n3. Select `postman/Spring-Boot-Scheduling-API.postman_collection.json`\n4. Import and start testing\n\n---\n\n## 🚀 Quick Start Testing\n\n### Step 1: Start the Application\n```bash\ncd SpringBootApplication\nmvn clean install\nmvn spring-boot:run\n```\n\n### Step 2: Wait for Sample Data Generation\n```\n========================================\nGenerating Sample Data for Testing\n========================================\n\n--- Generating Sample Reports ---\n✓ Created COMPLETED Report: ID=1, Name=Sales Report - Q1 2026\n✓ Created IN_PROGRESS Report: ID=2, Name=Customer Feedback Analysis\n✓ Created PENDING Report: ID=3, Name=Inventory Stock Check\n✓ Created FAILED Report: ID=4, Name=Database Migration Report\n...\n```\n\n### Step 3: Access H2 Console (Optional)\n```\nURL: http://localhost:8080/h2-console\nJDBC URL: jdbc:h2:mem:testdb\nUsername: sa\nPassword: (leave blank)\n```\n\n### Step 4: Test Using One of the Methods Below\n\n---\n\n## 📝 API Testing Methods\n\n### Method 1: Using cURL\n\n**Get all reports:**\n```bash\ncurl -X GET http://localhost:8080/api/reports\n```\n\n**Create a new report:**\n```bash\ncurl -X POST http://localhost:8080/api/reports \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"reportName\": \"Monthly Summary\",\n    \"generatedBy\": \"API\"\n  }'\n```\n\n**Get reports by status:**\n```bash\ncurl -X GET http://localhost:8080/api/reports/status/COMPLETED\n```\n\n**Update report status:**\n```bash\ncurl -X PUT http://localhost:8080/api/reports/1/status \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"status\": \"COMPLETED\",\n    \"content\": \"Report completed successfully\"\n  }'\n```\n\n### Method 2: Using Postman\n\n1. Import the collection\n2. Select endpoint from the list\n3. Click \"Send\"\n4. View formatted response\n\n### Method 3: Using JUnit Tests\n\n```bash\n# Run all tests with verbose output\nmvn test -X\n\n# Run tests and generate HTML report\nmvn surefire-report:report\n```\n\n### Method 4: Using Bash Script\n\n```bash\ncd scripts\nchmod +x test-api.sh\n./test-api.sh 2>&1 | tee test-results.log\n```\n\n---\n\n## 📊 Sample Test Scenarios\n\n### Scenario 1: Complete Report Lifecycle\n```bash\n# 1. Create report\ncurl -X POST http://localhost:8080/api/reports \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\"reportName\": \"Test Report\", \"generatedBy\": \"TEST\"}'\n\n# Response: {\"id\": 10, \"status\": \"PENDING\", ...}\n\n# 2. Update to IN_PROGRESS\ncurl -X PUT http://localhost:8080/api/reports/10/status \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\"status\": \"IN_PROGRESS\"}'\n\n# 3. Complete the report\ncurl -X PUT http://localhost:8080/api/reports/10/status \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\"status\": \"COMPLETED\", \"content\": \"Test content\"}'\n\n# 4. Verify completion\ncurl -X GET http://localhost:8080/api/reports/10\n```\n\n### Scenario 2: Task Execution Flow\n```bash\n# 1. Create task\ncurl -X POST http://localhost:8080/api/tasks \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\"taskName\": \"Test Task\", \"executionFrequency\": \"DAILY\"}'\n\n# 2. Start task\ncurl -X PUT http://localhost:8080/api/tasks/1/start\n\n# 3. Complete task\ncurl -X PUT http://localhost:8080/api/tasks/1/complete\n\n# 4. Verify task status\ncurl -X GET http://localhost:8080/api/tasks/1\n```\n\n### Scenario 3: Statistics Reporting\n```bash\n# Get report statistics\ncurl -X GET http://localhost:8080/api/reports/stats/count\n# Response: {\"PENDING\": 2, \"IN_PROGRESS\": 1, \"COMPLETED\": 5, \"FAILED\": 1}\n\n# Get task statistics\ncurl -X GET http://localhost:8080/api/tasks/stats/count\n# Response: {\"PENDING\": 1, \"IN_PROGRESS\": 0, \"COMPLETED\": 4, \"FAILED\": 1}\n```\n\n---\n\n## 🔍 Expected API Responses\n\n### Success Response (200
